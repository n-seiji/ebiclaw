# playbook/

このディレクトリには、司令塔(codex)が繰り返しの依頼に対応するための
手順書を置きます。`AGENTS.md` の手順1で参照される想定です。

## 書き方

- 1 ファイル = 1 手順。ファイル名は依頼内容が分かる名前にする
  (例: `bigquery-aggregation.md`)。
- 前提条件(必要な権限・ツール・環境変数)を先頭に書く。
- 手順は上から順に実行できる具体的なコマンド・クエリで書く。
- 出力の解釈方法や、よくある落とし穴があれば末尾に書く。

## サンプル

- `bigquery-aggregation.md` — BigQuery でのプロダクトデータ集計手順の例
